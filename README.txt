Desenvolvimento de Sistemas

Trabalho I


Sistema a ser desenvolvido:

O sistema a ser desenvolvido durante o semestre deve ser, ou o tema de Trabalho de Conclusão de um dos componentes da equipe, ou um dos sistemas sugeridos para a disciplina, ou uma proposta de novo sistema.

Se for o tema de Trabalho de Conclusão, a equipe deve entregar, além dos artefatos descritos a seguir, a proposta de TC.
Se for um dos sistemas propostos, em caso de empate, a prioridade será pela ordem de escolha.
Se for a proposta de um novo sistema, deve ser enviado pelo Moodle uma descrição do objetivo do sistema e dos requisitos iniciais identificados para análise antes do desenvolvimento do Trabalho I.
 

Escopo:

O processo padrão a ser utilizado para o projeto será o OpenUP. Se a equipe desejar utilizar outro processo de desenvolvimento de software deverá indicá-lo na avaliação do Trabalho I.

O escopo do Trabalho I envolve a fase de iniciação (Inception) do OpenUP, ou similar no processo escolhido pela equipe.

Desta forma, as atividades previstas e os artefatos a serem entregues devem seguir a descrição da fase do processo definido.

Tailoring de processo:

Além das atividades e artefatos previstos para a fase de iniciação, deve ser entregue também o Modelo de negócio, com os seguintes artefatos:

Visão do negócio:
Conforme template disponível ou Business Model Canvas
Modelo de negócio (modelagem):
Diagrama de casos de uso de negócio.
Diagrama de objetos de negócio (apenas do caso de uso de negócio que será automatizado pelo sistema)
Na execução do processo, considere os seguintes ajustes:

os artefatos do projeto devem ser organizados e entregues conforme a seguinte estrutura (pastas):
gerente de projeto
analista
arquiteto
desenvolvedor
testador
os nomes dos arquivos (ou pasta)  correspondentes a cada artefato devem ser indicados na planilha de avaliação e evidências.
o nome dos requisitos funcionais (casos de uso) deve possuir o formato UC<número>-<nome>. Exemplo: UC01-Fazer login.
para a descrição detalhada dos requisitos funcionais (casos de uso) utilizar o template: "Template de descrição detalhada de caso de uso", disponível no item 'Materiais de apoio'.
Para a fase de iniciação (do OpenUp) este template pode ser preenchido apenas até o item "Descrição". Os fluxos e demais seções seguintes poderão ser desenvolvidos nas próximas iterações
o nome dos casos de teste deve possuir o formato TC<número>-<nome>, que mapeie o caso de uso correspondente. Exemplos: TC01a-Fazer login com sucesso, TC01b-Fazer login com falha - usuário inválido.
Avaliação:

Nota do Trabalho = 10% apresentação + 90% documentação, sendo:
Apresentação: 50% presença pontual + 50% apresentação.
Documentação: 75% artefatos do papel individual + 25% média dos artefatos da equipe.
Entrega:

Todos os artefatos (arquivos) devem ser compactados em um único arquivo que deverá ser submetido via Moodle, através da tarefa de Entrega do Trabalho I.
O nome do arquivo deve ser constituído pelo primeiro nome dos componentes do grupo, seguido do dígito "1".
Exemplo: suponha que um grupo é constituído pelos alunos João da Silva, Maria de Souza e Paulo Moura, o arquivo deveria ser nomeado como: 'JoaoMariaPaulo1.zip'
A entrega deve ser realizada por cada membro da equipe, para controle de entrega do trabalho; mas será considerada a versão enviada pelo gerente do projeto.
